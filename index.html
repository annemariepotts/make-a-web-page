Lesson One: The Basics of the Web

World Wide Web

The world wide web is a collection of HTML (Hyper Text Markup Language) documents where hyperlinks connect documents. HTML is how you link various content to your webpage, including images and video.

Browsers, like Firefox and Chrome, make requests for webpages and documents via the internet to servers using HTTP. The servers, which are computers optimized for hosting files, respond with a file that the browser then displays.

Hyper Text Markup Language

In HTML, there are two primary categories - text content and mark up. Text content is what the visitor to the webpage will see, while mark up is the arrangement of that content. Mark up includes references to other documents. These documents can include images and videos. For instance, the Udacity classroom webpage has a reference to youtube video in the middle.

How to Think About A Website

Websites should be thought of like houses. The structure and walls of your house is the HTML. The decorations, carpeting, and paint is the CSS. Your interactive and changeable features, like the garage door opener, is Java. Just like in how families have rules for how their house functions, there are rules for the behavior of websites.

Elements of Websites

Elements in HTML are in a tree like structure, where you will have elements inside of other elements inside of other elements. This ever branching structure allows for complexity inside of a hierarchical structure that can be fairly easily read using the opening and closing tags of the elements.

Languages of Webpages

HTML and CSS are both languages. HTML controls the structure of a webpage, functioning as syntax and rules. The basic word of HTML is tag, < >, which the browser uses to create the elements in your tree, also called the "DOM", Document Object Model.

CSS controls the style and looks of a page. Using CSS, programmers can change how HTML elements look on the page. An HTML document uses a special section to define the CSS for the page, making it easier for the programmer in that you don't have to duplicate and replicate the same style setting for each individual block or element if they are similar. CSS automates the style for the programmer, after the initial set up and definitions.

The Internet is Made of Boxes

Everything on a webpage is a box, even things that look like circles or other shapes to the viewer! Why? Boxes allow for easy repositioning depending on browser size and user experience. Most webpages will have boxes inside of other boxes with the larger boxes being key features like the header, the middle, and featured hyperlinks to other content.

Why is Repetition a Bad Thing?

Although perfect practice makes perfect, in programming, repetition creates more work, and opportunities for validation errors, in the long run. Instead of having to style each individual paragraph or thought in HTML, CSS allows the programmer the freedom to style many like objects at the same time. In case of future design change for the website, the programmer who uses CSS will only need to make changes to the styling and decorations instead of the structure and content of the site. These style changes will also impact the entire document simultaniously, rather than the programmer needing to go element by element to make style changes.

Repetition increases the chances of careless errors. By using CSS to style websites, programmers can make drastic changes to the looks of the website more simply and without jeopardizing the structure.
